# Roadmap

## Immediate fixes
- Clean frontend lint warnings blocking production builds.
- Remove remaining dependency/security audit findings through safe upgrades.
- Add baseline smoke tests for frontend routes and API health.
- Confirm real deployment target and environment variables.

## Short-term roadmap
- Add Arabic/English SEO metadata per page.
- Add sitemap, robots.txt, canonical URLs, and LegalService schema markup.
- Standardize package manager usage.
- Add contact/payment validation and rate limiting.
- Create deployment runbooks for staging and production.

## Mid-term roadmap
- Consolidate backend architecture around Laravel or Node with migration plan.
- Add admin audit logs and role-based access controls.
- Add E2E tests for consultation inquiry, service browsing, checkout, and admin content flows.
- Add monitoring, error tracking, uptime checks, and backup automation.

## Long-term roadmap
- Build secure client portal and matter intake workflows.
- Add knowledge base, legal service calculators, and document request flows.
- Expand to GCC-ready localization, payment methods, and compliance documentation.

## Product roadmap
- Improve service packaging, lead capture, consultation booking, and trust signals.
- Add content taxonomy by legal practice area and client segment.
- Add conversion analytics and funnel reporting.

## Engineering roadmap
- CI/CD hardening, dependency automation, test coverage, API contracts, performance budgets, and release automation.

## AI roadmap
- Plan RAG-ready legal content assistant with human review, citation controls, hallucination safeguards, and legal disclaimer UX.

## LegalTech feature roadmap
- Secure intake forms, client dashboards, appointment booking, document upload, case/matter tracking, e-sign readiness, and compliance logs.

## Growth roadmap
- Arabic-first SEO content clusters, landing pages by legal service, case-study/social-proof framework, newsletter, and analytics-driven experiments.
