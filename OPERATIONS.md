# Operations

## Daily operations
Check uptime, error logs, contact submissions, payment health, admin activity, analytics anomalies, and urgent client requests.

## Weekly operations
Review dependency alerts, content quality, SEO performance, backups, unresolved support tickets, and roadmap progress.

## Monthly operations
Review security posture, access lists, incident drills, performance metrics, conversion funnel, legal content plan, and compliance evidence.

## Monitoring
Use uptime checks, application errors, API latency, payment failures, form conversion, and security alerts.

## Backups
Back up databases and uploaded documents with encrypted, tested restoration procedures.

## Incident response
Classify severity, assign owner, communicate status, contain, remediate, validate, and document lessons learned.

## Support workflow
Capture inquiry, classify legal/service/technical issue, assign owner, respond within SLA, close with notes, and track recurring issues.

## Release workflow
Plan scope, update docs/version/changelog, validate staging, approve, deploy, smoke test, monitor, and document release notes.

## Maintenance workflow
Patch dependencies, rotate secrets, review logs, update docs, prune obsolete content, and improve tests.

## Ownership matrix
- Product: roadmap and acceptance criteria.
- Engineering: code, architecture, tests, CI/CD.
- Security/Compliance: secrets, PDPL, access, incidents.
- Marketing/Growth: SEO, content, analytics, conversion.
- Operations: support, monitoring, releases, runbooks.
