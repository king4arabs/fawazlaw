## Production Build

### Frontend (`UI-WEB/frontend`)
```bash
cd UI-WEB/frontend
npm ci --no-audit --no-fund
CI=true npm test -- --watchAll=false --passWithNoTests
CI=false npm run build
```
- Output directory: `UI-WEB/frontend/build/` — static assets ready for any static host (Vercel, Netlify, Cloudflare Pages).
- `CI=false` is intentional and tracked in `TODO.md`; it tolerates pre-existing CRA lint warnings without masking real errors.

### Node API (`API-Node`)
```bash
cd API-Node
npm install --no-audit --no-fund
npm audit --audit-level=moderate || true
node -c index.js
```
- This component remains a prototype. Deploy only if you actively rely on it; the Laravel API is the primary backend.

### Laravel API (`API-BACKEND/api`)
```bash
cd API-BACKEND/api
composer install --no-dev --optimize-autoloader
cp .env.example .env   # only on first deploy
php artisan key:generate --force   # only on first deploy
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```
- PHP CI is still not configured (see `TODO.md`); run the above locally or on the deploy host and verify Swagger (`/api/documentation`) and Sanctum endpoints respond before promoting.

## Deployment (fawazlaw.sa)

Follow `docs/runbooks/release.md` and the checklists in `DEPLOYMENT.md`. Summary:

1. **Pre-deployment** — verify CI green on the release commit, `VERSION.md`/`CHANGELOG.md` updated, `npm audit`/`composer audit` reviewed, backups taken in the last 24h.
2. **Cut** — merge the release PR, then tag:
   ```bash
   git tag v0.3.0 -m "v0.3.0 — Production Build & Deployment"
   git push origin v0.3.0
   ```
3. **Deploy frontend first** — push the `UI-WEB/frontend/build/` artifact (or trigger the host's build) to the apex `fawazlaw.sa`. Verify HTTPS, HSTS, canonical URLs, `sitemap.xml`, `robots.txt`, manifest.
4. **Deploy APIs only if changed** — this release has no API source changes, so API redeploys are optional. If you redeploy anyway, do `api.fawazlaw.sa` (Laravel) before any consumer that depends on it.
5. **Post-deployment** — run the post-deployment checklist in `DEPLOYMENT.md`: API health, payment sandbox transaction, Lighthouse mobile pass, error tracker, contact form, WhatsApp link, app portal link.
